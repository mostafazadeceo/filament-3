# 0000 — Record Architecture Decisions

## Context and Problem Statement
پروژه دارای تعداد زیادی ماژول، سیاست‌ها و وابستگی‌های بین‌ماژولی است. بدون ثبت تصمیمات معماری، انتقال دانش دشوار و ریسک انحراف طراحی بالا می‌رود.

## Considered Options
- ثبت تصمیمات در ADR (فرمت MADR)
- نگهداری تصمیمات در READMEهای پراکنده
- بدون ثبت رسمی تصمیمات

## Decision Outcome
ADR با فرمت MADR برای تمام تصمیمات کلیدی معماری انتخاب شد و در این مسیر ذخیره می‌شود.

## Consequences
- تصمیمات کلیدی قابل ردیابی و به‌روزرسانی خواهند بود.
- هر تغییر معماری باید ADR جدید یا اصلاح ADR موجود داشته باشد.
