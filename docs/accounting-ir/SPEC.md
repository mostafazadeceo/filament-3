# Accounting IR — SPEC

## Product Goals
- جامع‌ترین سیستم حسابداری ایران با معماری ماژولار و API-First.
- سازگار با Filament v4، چندشرکتی/چندشعبه‌ای/چندتننتی.
- انطباق کامل با قوانین ایران: VAT، مالیات مستقیم، 169، سامانه مؤدیان.
- ثبت غیرقابل‌دستکاری برای دفاتر قانونی و ردپای حسابرسی.
- تجربه کاربری فارسی، RTL، جلالی، اعداد فارسی.

## Non-Goals
- ابزارهای پایش/نظارت تهاجمی کاربر (No surveillance).
- تغییرات گسترده در هسته اپلیکیشن میزبان.

## Personas & Needs
- مدیر مالی/حسابدار ارشد: دقت، صحت، کنترل داخلی، گزارش‌های قانونی.
- حسابدار/کاربر عملیاتی: ورود سریع داده، خطایابی قابل فهم، گردش کار تأیید.
- حسابرس/بازرس: ردپای تغییرات، عدم‌ویرایش دفاتر، گزارش‌های قابل اتکا.
- مدیرعامل/مدیر پروژه: داشبوردهای مدیریتی، جریان نقدینگی و سود پروژه.

## Domain Model
- شرکت/شعبه/سال مالی/دوره مالی
- کدینگ حساب‌ها (گروه/کل/معین/تفصیلی) + پلن کدینگ
- ابعاد حسابداری (مرکز هزینه/پروژه/شعبه/قرارداد/پرسنل/محصول)
- اسناد حسابداری و خطوط سند + پیوست + تاییدات
- اشخاص/مشتریان/تامین‌کنندگان + پروفایل مالیاتی
- کالا/خدمت + واحد اندازه‌گیری + دسته مالیاتی
- فروش/خرید + پرداخت/دریافت + تخصیص‌ها
- خزانه (بانک/صندوق) + تراکنش‌ها + چک‌ها
- انبار + رسید/حواله + گردش کالا
- دارایی ثابت + برنامه استهلاک
- حقوق و دستمزد + جداول نرخ‌های نسخه‌دار
- پروژه/پیمانکاری + صورت وضعیت + کسورات
- مالیات‌ها: VAT، تکلیفی، 169
- سامانه مؤدیان: صورتحساب الکترونیکی + Provider/Adapter
- Integration Hub + Mapping + Logs
- Audit Events (غیرقابل حذف)

## Data Architecture
- همه جداول با `tenant_id` و در صورت نیاز `company_id`.
- Soft delete برای داده‌های مرجع؛ اسناد مالی قطعی قابل حذف نیستند.
- ایندکس‌های اصلی: tenant_id, company_id, status, dates, party_id, project_id.
- نرخ‌ها و جداول قانونی نسخه‌دار: `effective_from/effective_to`.

## Security Model
- Spatie Permission (teams) + IAM Suite.
- Policy و Scope برای تمام Resourceها و APIها.
- مجوزهای granular (view/manage/approve/post/lock/export/send).
- رویدادهای حساس ثبت Audit غیرقابل حذف دارند.

## Workflow Engine (Accounting)
- چرخه سند: draft → submitted → approved → posted → locked.
- ویرایش سند قطعی ممنوع؛ اصلاح فقط با سند معکوس.
- قفل دوره مالی و سال مالی.

## Iran Compliance
- VAT دوره‌ای با نرخ‌های نسخه‌دار.
- مالیات تکلیفی برای پرداخت‌ها.
- گزارش معاملات فصلی (169/169 مکرر) با Template قابل بروزرسانی.
- سامانه مؤدیان با معماری Adapter-based و نسخه‌بندی دستورالعمل‌ها.

## Extensibility
- Domain Services + DTOs پایدار.
- Adapterها برای E-Invoice Providers و Integration Connectors.
- Event Bus برای publish/subscribe ماژول‌های دیگر.

## API
- مسیر پایه: `/api/v1/accounting-ir/...`
- احراز هویت: ApiKeyAuth + ApiAuth + ResolveTenant
- Rate limit از config
- OpenAPI با مسیر `/api/v1/accounting-ir/openapi`

## Roadmap
- MVP: هسته حسابداری، کدینگ، اسناد، فروش/خرید، خزانه.
- Phase 2: مالیات‌ها، 169، انبار پیشرفته، دارایی ثابت، حقوق.
- Phase 3: سامانه مؤدیان کامل، Integration Hub، گزارش‌های پیشرفته.
