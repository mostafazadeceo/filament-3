# Commerce Core Specification

## Scope
Commerce core provides the shared domain kernel for catalog, pricing, inventory, customers, and compliance events. It is tenant-scoped and UI/API permission-gated.

## Domain entities
- Product, Variant, Category, Brand
- Price List, Price
- Inventory Item, Stock Move
- Customer
- Compliance Exception, Fraud Rule, Compliance Digest

## Key behaviors
- Catalog and pricing support multi-site and multi-currency.
- Inventory supports adjustments and stock moves with references.
- Compliance exceptions are generated via rules and resolved with audit metadata.

## Tenancy and authorization
- All models use BelongsToTenant and are scoped in UI and API.
- Permissions are registered under `commerce.*` capability groups.

## Data integrity
- Unique constraints for tenant-scoped identifiers (slug, code).
- Indexes on status, tenant_id, and updated_at for high-volume queries.

## Compliance
- Exceptions are created by CommerceComplianceService based on rules and thresholds.
- Weekly digests are generated by a command and dispatched via notify-core.
