# فرضیات

- پروژه از Laravel 11.28+ و Filament v4 استفاده می‌کند.
- جدول کاربران اصلی وجود دارد و می‌توان ستون‌های امنیتی را به آن افزود.
- مدل کاربر از Traitهای `HasIamSuite`، `HasRoles` و `HasApiTokens` استفاده می‌کند تا نشست‌ها قابل مدیریت باشند.
- اسکوپ تننت با هدر `X-Tenant-ID` یا کانتکست پنل فعال می‌شود.
- در صورت استفاده از توکن‌های Sanctum، توانایی‌ها (`abilities`) تعریف می‌شوند و می‌توان از قالب `tenant:{id}` برای اسکوپ تننت استفاده کرد.
- برای چندتننتی در Spatie Permission، گزینه `teams` فعال و `team_foreign_key=tenant_id` تنظیم می‌شود.
- صف و زمان‌بند لاراول فعال است و Jobها قابل اجرا هستند.
- برای ارسال اعلان/رمز یکبارمصرف یک پلاگین موجود است که از طریق مبدل متصل می‌شود.
- اگر پنل سوپرادمین دارید، شناسه پنل در `filamat-iam.super_admin_panels` تعریف می‌شود.
- کنترل دسترسی مبتنی بر اشتراک فعال است و در صورت نبود اشتراک فعال، فقط مجوزهای معاف از کانفیگ عبور داده می‌شوند.
- امپرسونیشن تنها توسط سوپرادمین و در پنل کلان انجام می‌شود.
